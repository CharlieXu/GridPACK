35 /// Enapsulation of a ParMETIS format graph
36 /**
37  * The main reason for this class is to be able to have a ParMETIS
38  * representation of a graph whose distribution is different than the
39  * actual, current distribution (as represented by an AdjacencyList
40  * instance).
41  *
42  * ParMETIS requires that at least one of the graph nodes exists on
43  * each processor. This class, hopefully, will be used to overcome
44  * that limitation, so, for examples, graphs that exist on only one
45  * processor can be partitioned and redistributed
46  *
47  * In addition, AdjacencyList works with arbitrarily ordered global
48  * node indices. ParMETIS requires special ordering of nodes. This
49  * class is used to translate between the two indexing systems. In
50  * both cases, the global node indices must start with zero.
51  */
