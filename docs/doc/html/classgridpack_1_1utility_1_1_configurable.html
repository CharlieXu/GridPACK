Detailed Description

This serves as the base for any class that has run-time options. The options required by a instance of this base class are identified by a string and may have any (or one of many) types.

Instances have two paths, one for default values for all instances and one for specific instances. The instance is created with the former and it cannot be changed by any public means. The latter may be created created by prepending a name or path to the default. For example, suppose a "linear solver" is a subclass, and it needs a "tolerance". The default path might be:

"linear solver"/"tolerance" = 1.0e-06

which would be used by all instances. Say there is "nonlinear solver" that uses a "linear solver" instance (both subclasses of Configurable), a path such as

"nonlinear solver"/"linear solver"/"tolerance" = 1.0e-03

would distinguish it from the default.

I think this is just a wrapper on boost::property_tree.
